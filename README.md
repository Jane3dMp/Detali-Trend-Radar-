# Trend Radar для «Детали» — v0.2

Сборщик сигналов о трендах мастер-классов для взрослой аудитории.
Опережаем городских конкурентов, подсматривая в рынки, которые идут на 6–12 месяцев впереди.

## Что изменилось по сравнению с v0.1

| Было (v0.1) | Стало (v0.2) |
|---|---|
| Google Trends + Eventbrite | Eventbrite + Timepad (Google Trends убран) |
| 4 города (NY, London, Berlin, LA) | 2 западных (London, Berlin) + 2 российских (Москва, СПб) |
| CSV-файлы | SQLite + CSV-экспорт |
| Локальный запуск | GitHub Actions (cron каждое воскресенье) |
| Один signal_score | signal_score с учётом «окна возможности» (есть на западе, нет в РФ) |

## Установка и запуск

```bash
pip install -r requirements.txt

# Тестовый прогон (3 ключевика, 1 город, ~2 минуты)
python -m trend_radar.main --test

# Полный прогон (~15–25 минут)
python -m trend_radar.main

# Только экспорт последнего прогона в CSV
python -m trend_radar.main --export
```

## Как считается signal_score

Формула учитывает не просто популярность, а **окно возможности**: тренд, который горячий
на западе и холодный в России, получает максимальный балл.

| Компонент | Вес | Логика |
|---|---|---|
| Западные события | 25 | Больше событий на Eventbrite → выше |
| Западная цена | 15 | Дороже билет → маржинальнее ниша |
| Российские события | 15 | Больше событий на Timepad → тренд пришёл |
| **Окно возможности** | **25** | **Много на западе × мало в РФ → максимум** |
| Ценовая премия | 10 | Дорогой западный чек → платёжеспособная аудитория |
| Бонус разнообразия | 10 | Только на западе = полный бонус; обе платформы = полбонуса |

Пример на реальных данных:

| Ключевик | Запад | Россия | Score | Почему |
|---|---|---|---|---|
| tufting workshop | 25 событий, £95 | 2 события | **65.9** | Горячий на западе, почти не пришёл |
| paint and sip | 35, £65 | 15 событий | 61.0 | Популярен, но уже в Москве |
| pottery class | 40, £55 | 25 событий | 56.5 | Коммодити, окно закрыто |

## Настройка GitHub Actions

1. Создайте приватный репозиторий на GitHub.
2. Запушьте весь проект.
3. Файл `.github/workflows/collect.yml` уже готов — Actions запустится
   автоматически каждое воскресенье в 21:00 UTC (00:00 по Минску).
4. Ручной запуск: вкладка Actions → «Trend Radar — Weekly Collection» → «Run workflow».
5. Результаты коммитятся обратно в репозиторий (trends.db + CSV + логи).
